# Contributing Guide

## How to Contribute
- Use clear commit messages.
- Create branches for major changes.
- Submit pull requests with explanations.

## Coding Style
- Python: PEP 8  
- Markdown: clean headings, short paragraphs  
- Folder names: lowercase_with_underscores  

## Documentation Rules
- Every new feature must include a `.md` explanation.
- Diagrams go into `/media` or `/docs/images`.

## Branching Model
- `main` → stable  
- `dev` → active development  
- `feature/*` → new features
